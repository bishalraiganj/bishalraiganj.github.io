0x01.B Jun 2026 – Present distributed systems · fintech

Distributed Banking Transaction & Ledger System

JavaSpring BootApache KafkagRPCMicroservicesSaga ChoreographyTransactional OutboxSpring Data JPAMySQL

Money movement is the hardest distributed-systems problem there is: every transfer must be exactly-once in effect, auditable, and consistent across services that can each fail independently. This platform is my answer, built with production-grade patterns.

  • Spring Boot microservices communicating over gRPC (HTTP/2) for low-latency internal calls and Apache Kafka for async, replayable event flows.
  • Transactional Outbox Pattern for atomic "write + publish" — no lost events, no dual-write inconsistencies.
  • Saga Choreography for distributed transactions: services react to each other's events, with compensating actions instead of fragile distributed locks.
  • Concurrent Outbox Relay jobs with idempotent Kafka publishing — safe under retries, parallel by design.
  • Transfer and ledger modules built on Spring Data JPA + MySQL, keeping the books balanced down to the row.

Smart Log Analytics & Alerting System

JavaForkJoin PoolSpring BootMySQLREST API
